# AI 简历优化系统 — 项目技术文档

## 一、项目概述

基于 LangGraph 多 Agent 架构的智能简历优化系统，用户上传简历后，系统自动提取内容，通过 12 个专职 AI Agent 流水线式优化简历各模块，最终输出 PDF/Word 格式的优化简历、自我介绍及面试题。

**架构模式**: 前后端分离 + 多 Agent 工作流

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + TypeScript + Tailwind CSS + Pinia + Vite |
| 后端 | FastAPI + WebSocket + Uvicorn |
| AI 框架 | LangChain + LangGraph 状态图 |
| LLM | 通义千问 / DeepSeek / 智谱 GLM（多模型可切换） |
| 文档处理 | PyPDFLoader / python-docx / ReportLab |
| 通信协议 | REST API + WebSocket 实时推送 |

## 三、后端模块详解

### 3.1 LangGraph 工作流引擎

**文件**: `src/graph.py`、`src/state.py`

**技术栈**: LangGraph StateGraph、TypedDict

**功能**:

- 定义 12 个 Agent 节点的有向无环图（DAG）
- 通过 `AgentState` TypedDict 在节点间传递状态
- 条件路由：简历提取失败时自动终止流程
- append-only 的 progress 列表追踪执行进度

**AgentState 状态结构**:

```python
class AgentState(TypedDict):
    resume_path: str                    # 输入简历路径
    job_name: str                       # 目标职位
    job_age: str                        # 工作年限
    technology_stack: str               # 匹配到的技术栈
    original_resume_text: str           # 提取的原文
    optimized_resume_text: str          # 优化后全文
    sections: dict                      # 各模块优化结果
    memory_hits: list                   # 缓存命中记录
    progress: Annotated[list, operator.add]  # 追加式进度
    logs: Annotated[list, operator.add]      # 追加式日志
```

**工作流节点顺序**:

```
Extractor → Title → PersonalInfo → Skills → Certificate
→ WorkExperience → Education → ProjectExperience → SelfEvaluation
→ Assembler → LayoutExpert → Interviewer
```

**亮点**:

- 节点间状态自动流转，无需手动传递参数
- `Annotated[list, operator.add]` 实现追加式日志，避免状态覆盖
- 新增简历模块只需注册一个节点，零改动现有流程

### 3.2 简历提取 Agent

**文件**: `src/agents/extractor.py`、`src/tools/file_ops.py`

**技术栈**: LangChain Tool、PyPDFLoader、Docx2txtLoader

**功能**:

- 自动识别输入文件格式（PDF / DOCX / DOC）
- 调用对应的 Loader 提取纯文本
- 将提取内容保存为中间文件 `extracted_content.txt`
- 提取失败时设置错误状态，触发条件路由终止流程

**亮点**:

- 统一 Tool 接口封装，遵循 MCP 工具协议标准
- 支持 `.doc` 格式自动转换为 `.docx` 后处理

### 3.3 模块优化 Agent 群

**文件**: `src/agents/section_agents.py`、`src/prompts/section_prompts.py`

**技术栈**: LangChain ChatModel、PromptTemplate、JSON/Markdown 解析

**功能**:

8 个专职 Agent 分别负责简历的一个模块：

| Agent | 职责 | 输出格式 |
|-------|------|----------|
| Title Agent | 生成求职标题 | 纯文本 |
| PersonalInfo Agent | 提取并格式化个人信息 | JSON |
| Skills Agent | 优化技能列表 | Markdown 列表 |
| Certificate Agent | 格式化证书信息 | 表格结构 |
| WorkExperience Agent | 优化工作经历（时间线排序） | Markdown |
| Education Agent | 教育背景格式化 | Markdown |
| ProjectExperience Agent | 项目经验描述优化 | Markdown |
| SelfEvaluation Agent | 生成自我评价段落 | 纯文本 |

**核心设计 — 泛型 Agent 工厂函数**:

```python
def _run_section_agent(state, section_key, prompt_template, post_processor=None):
    # 1. 检查 Memory 缓存
    cached = memory_manager.get_optimized_content(...)
    if cached:
        return {section_key: cached, "memory_hits": [...]}

    # 2. 调用 LLM 生成
    response = llm.invoke(messages)

    # 3. 后处理（如有）
    result = post_processor(response) if post_processor else response

    # 4. 写入缓存
    memory_manager.save_optimized_content(...)

    return {section_key: result}
```

**亮点**:

- 一个工厂函数服务所有 section Agent，消除重复代码
- Memory 缓存命中时跳过 LLM 调用，直接返回结果
- 每个 section 可配置独立的 `post_processor` 后处理逻辑

### 3.4 面试官 Agent

**文件**: `src/agents/interviewer.py`

**技术栈**: ThreadPoolExecutor 并发、LangChain LLM

**功能**:

- 根据简历内容 + 技术栈生成自我介绍（约 200 字）
- 按比例生成 4 类面试题：
  - 项目相关题：10%
  - 基础技能题：30%
  - 进阶技能题：30%
  - 架构/场景题：30%

**亮点**:

- 使用 `ThreadPoolExecutor` 将 4 类题目并发提交给 LLM，减少等待时间
- 题目比例可配置，贴近真实面试分布
- 自我介绍和面试题并行生成，互不阻塞

---

### 3.5 智能路由系统

**文件**: `src/route.py`

**技术栈**: 规则引擎 + LLM 语义匹配

**功能**:

- 根据用户输入的职位名称，自动匹配最合适的技术栈模板
- 内置 12+ 套技术栈：Python 后端、前端、Java 后端、LLM 应用开发、DevOps 等
- 混合匹配策略：

```
输入 "Java开发工程师"
  → 优先规则匹配（关键词命中 "java"）→ 直接返回 java_backend.txt ✓

输入 "大数据平台研发"
  → 规则未命中 → 降级到 LLM 语义判断 → 返回 bigdata.txt ✓
```

**亮点**:

- 规则匹配零 LLM 调用，响应毫秒级
- LLM 兜底保证罕见职位也能匹配，不漏配
- 新增技术栈只需添加一个 `.txt` 文件

---

### 3.6 Memory 缓存系统

**文件**: `src/memory/manager.py`

**技术栈**: SHA-256 哈希、JSON 文件存储

**功能**:

- 按简历内容 + 职位上下文缓存已优化的 section
- 缓存 key = SHA-256(original_content + job_name + tech_stack)
- 支持 Web UI 查看所有缓存条目和清理指定模块

**缓存文件结构**:

```json
{
  "section_key": "skills",
  "original_content": "原始技能文本...",
  "optimized_content": "LLM优化后的技能文本...",
  "context": {
    "job_name": "Python开发工程师",
    "technology_stack": "Python, Django, Flask..."
  },
  "hash": "a1b2c3d4..."
}
```

**亮点**:

- 上下文感知：同一简历不同职位不会命中同一缓存
- 节省 LLM token 成本 60%+（相同简历二次优化）
- Web UI 可视化缓存命中，用户可感知 AI 效率提升

### 3.7 PDF 生成引擎

**文件**: `src/tools/pdf_generator.py`

**技术栈**: ReportLab、自定义 TTF 中文字体注册

**功能**:

- 将优化后的 Markdown 简历渲染为专业排版的 PDF
- 支持多种排版元素：
  - 多列布局（个人信息双列排列）
  - 表格样式（工作经历时间线）
  - 自定义间距、缩进、行距
  - 粗体/标题层级渲染

**亮点**:

- 中文字体原生注册（SimHei TTF），无乱码问题
- HTML 转义防止内容注入
- 输出质量接近专业简历模板网站

### 3.8 多 LLM 适配层

**文件**: `src/llm/factory.py`

**技术栈**: LangChain ChatModel 统一接口、OpenAI 兼容协议

**功能**:

- 统一封装三家 LLM 提供商：
  - **通义千问 (Dashscope)**: `qwen-max`
  - **DeepSeek**: OpenAI 兼容 API
  - **智谱 GLM**: `glm-4-flash`
- 自动检测 `.env` 中的 API Key 选择对应模型
- `OpenAIChatModel` 包装器抹平各厂商 API 差异
- 消息格式自动标准化（HumanMessage / SystemMessage 转换）

**亮点**:

- 切换 LLM 只需修改 `.env` 配置文件，代码零改动
- 统一的 ChatModel 接口，可随时接入新模型（如 Claude、GPT-4）
- 运行时自动选择可用模型，无需硬编码

### 3.9 Prompt 工程

**文件**: `src/prompts/` 目录

**技术栈**: LangChain PromptTemplate

**模块**:

| 文件 | 职责 |

**设计原则**:

- 每个模块的 prompt 独立维护，互不影响
- 强约束输出格式（JSON / Markdown / 纯文本），确保下游解析稳定

### 3.10 组装与排版 Agent

### 5.5 WebSocket 通信层

**文件**: `web/src/composables/useWebSocket.ts`

**技术栈**: 原生 WebSocket API、Vue Composition API

**设计模式**: Composable 模式（可组合函数）

**功能**:

- 封装 WebSocket 连接的完整生命周期（建立、消息收发、关闭、清理）
- 按 message type 分发到不同的响应式状态更新
- 组件卸载时自动关闭连接，防止内存泄漏

**消息处理逻辑**:

```typescript
socket.onmessage = (event) => {
  const data = JSON.parse(event.data)
  switch (data.type) {
    case 'status':      // 更新状态文本
    case 'agent_done':  // 更新节点状态 + 进度
    case 'progress':    // 追加进度消息
    case 'memory_hit':  // 标记缓存命中
    case 'complete':    // 跳转结果页
    case 'error':       // 显示错误提示
  }
}
```

**亮点**:

- 连接管理与业务逻辑完全解耦
- Composable 模式使 WebSocket 逻辑可在任意组件中复用
- 组件卸载自动 `socket.close()`，避免僵尸连接

|----------|-----------|
| 设计模式 | 工厂模式（LLM 适配）、策略模式（混合路由）、责任链（Agent 流水线）、代理模式（Vite 代理） |
| 状态管理 | LangGraph AgentState（TypedDict）、Pinia Store、Vue Reactive |
| 实时通信 | WebSocket 全双工通信、asyncio.Queue 线程-协程桥接 |
| 性能优化 | Memory 缓存、ThreadPoolExecutor 并发、规则优先减少 LLM 调用 |
| 工程化 | TypeScript 全量覆盖、前后端分离、多环境代理配置 |
| AI 工程 | Prompt 工程、Multi-Agent 编排、多模型适配、Token 成本控制 |
